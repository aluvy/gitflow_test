Git Flow 란?

Git Flow는 원래 깃 브랜치 관리를 위한 전략이었던 Git Workflow다.

Git Flow는 feature 브랜치와 여러 기본(primary) 브랜치를 사용하는 대한적인 Git 브랜치 모델이다.

이 모델에서 개발자는 feature 브랜치를 만들고 기능이 완료될 때까지 이를 기본 트렁크 분기에 병합(merge)하는 것을 연기한다.

Git Flow는 예정된 출시 주기가 있는 프로젝트와 지속적 제공의 DevOps 모범 사례에 사용할 수 있다.

Develop Branch

main branch 에서 생성한다.

Git Flow는 main branch와 develop branch 2개의 branch를 기준으로 하여, side branch를 생성하여 소스 관리를 한다.

side branch에는 feature, release, hotfix branch가 있다.
